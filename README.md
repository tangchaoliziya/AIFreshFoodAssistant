# AI 社区餐桌预测引擎 · 即时烹饪场景经营平台

> **一句话定位**：我们让超市不再被动清库存，而是用 AI 把今天卖不动的商品，变成顾客今晚想买的一顿饭。

这是面向"AI + 零售"公司内部路演的完整选题交付物，包含一份产品 Spec 文档与一个可现场演示的交互式 Web Demo。

## 🚀 快速开始

### 打开 Demo

**方式一（推荐，支持扫码功能）**：在 `demo/` 目录下启动本地服务器：

```bash
python -m http.server 8765
```

然后用浏览器访问 `http://localhost:8765/`。

> ⚠️ **扫码功能需通过局域网IP访问**：若要手机扫码打开菜谱页，请用电脑的局域网IP（如 `http://192.168.x.x:8765/`）代替 `localhost` 访问，手机与电脑需在同一WiFi下。

**方式二（快速预览，不支持扫码）**：直接双击 `demo/index.html` 打开。

- **全屏体验**：建议按 `F11` 全屏，投影效果最佳。

### 基本操作

1. **顶部场景切换**：点击「🌧️ 雨天晚餐」「☀️ 高温清凉」「👨‍👩‍👧 周末家庭」「🥟 冬至节日」切换演示场景。
2. **生成今日方案**：点击右上角橙色按钮，AI 推理链动画展开，自动生成菜单 / 场景包 / 触达内容 / 价值预估。
3. **查看菜谱**：点击任意菜谱卡片，弹出完整菜谱详情（食材分量 + 制作步骤 + 小贴士），底部含二维码。
4. **扫码分享**：用手机扫描菜谱弹窗中的二维码，可在手机上打开完整菜谱页面（`recipe.html`），方便收藏和分享。
5. **触达矩阵**：在右侧「全渠道触达矩阵」中切换「会员App / 门店大屏 / 员工指令 / 陈列图」查看多端输出。
6. **重新生成**：再次点击按钮可重新播放推理动画。

## 🎯 路演演示剧本（5-7 分钟）

### 开场（30 秒）

> "传统超市面对临期菜只有一个动作——打折。今天我演示的，是另一种思路：AI 把今天卖不动的商品，变成顾客今晚想买的一顿饭。"

### 第一步 · 看痛点（30 秒）

- 滚动到底部「价值预估」的对比图（先不生成，直接讲传统打折 vs 场景化的差异）。
- 点出核心观点：**卖不动的不是商品，是"组织方式"。**

### 第二步 · 看输入（1 分钟）

- 指向左侧「今日态势感知」：下雨、青椒 1 天临期、猪肉库存偏高、米饭熟食销量降、家庭客群 18:00 下班。
- 强调：这些数据超市本来就有，只是没有被"组织"起来。

### 第三步 · 看 AI 思考（1.5 分钟）⭐ 重点

- 点击右上角「生成今日方案」。
- 逐条讲解 AI 推理链：
  1. 青椒临期 + 猪肉高库存 → 匹配青椒肉丝
  2. 豆腐临期 → 匹配麻婆豆腐
  3. 雨天热菜需求 + 米饭销量降 → 联动熟食区
  4. 家庭客群 + 下班高峰 → 2 人份备菜包，17:30 推送
  5. 组合优化 → 品类均衡、毛利达标

### 第四步 · 看输出（2 分钟）⭐ 重点

- **推荐菜单**：中间区域，4 道菜的卡片，标注了哪些用了临期食材（红色标签）。
- **💡 点击菜谱卡片**：弹出完整菜谱（食材分量 + 制作步骤 + 小贴士），底部有二维码——**现场用手机扫码即可打开菜谱页面**，展示"从库存到餐桌"的完整闭环。
- **场景包**：右侧上方，套餐组合、定价、联动加购。
- **触达矩阵**：右侧下方，逐个切换展示——
  - 「会员App」：模拟手机推送卡片
  - 「门店大屏」：模拟门店屏幕菜谱
  - 「员工指令」：拣货 / 陈列 / 熟食备餐任务
  - 「陈列图」：门店场景化陈列布局
- 强调：**一个 AI 决策，全渠道同步落地。**

### 第五步 · 看价值（1 分钟）

- 滚动到底部「价值预估」：减损 ¥2,840、客单价 +18%、熟食客流 +35%、会员打开率 +27%。
- 对比图：传统打折 5 折 vs 场景化 8 折——**保住毛利的同时清掉库存。**

### 第六步 · 看扩展（1 分钟）

- 切换到「☀️ 高温清凉」场景，重新生成。
- 展示完全不同的输出（凉拌 + 解暑 + 写字楼午间双时段）。
- 再快速切「👨‍👩‍👧 周末家庭」「🥟 冬至节日」。
- 强调：**同一个引擎，不同场景输出差异化方案——这就是"经营大脑"。**

### 收尾（30 秒）

> "传统思路是'这个菜卖不动，打折'。我们的思路是'这个菜卖不动，AI 把它变成一道今晚想吃的菜'。未来超市不是卖货架，而是实时生成附近社区的餐桌需求。这就是我们想做的 AI 零售经营大脑。"

## 💡 核心差异化

| 维度 | 传统清库存 | AI 补货 / 优惠券 | **本产品** |
|------|-----------|-----------------|-----------|
| 逻辑 | 打折清掉 | 预测需求发券 | 把库存重组成消费场景 |
| 输出 | 打折标签 | 券 / 补货单 | 菜谱 + 场景包 + 全渠道触达 |
| 顾客视角 | "便宜的临期菜" | "一张券" | "今晚 15 分钟做青椒肉丝" |
| 解决痛点 | 单一（损耗） | 单一（销量） | 六合一 |
| 故事性 | 弱 | 中 | 强（有画面、有温度） |

## 📐 技术说明

- **Demo 版**：纯前端单页应用，预置 4 个场景 + 16 道完整菜谱，规则引擎模拟 AI 推理。确保路演 100% 稳定。
- **菜谱与二维码**：菜谱数据共享于 `data.js`；点击菜谱卡片弹出详情弹窗，内含纯 JS 生成的二维码（零依赖，无需联网）；扫码可打开 `recipe.html` 手机端菜谱页，支持收藏和分享。
- **V1 真实版**（规划）：React/Vue 前端 + Node/Python 后端，接入真实库存 ERP、天气 API、客流 IoT；AI 层采用 LLM（菜谱/文案生成）+ 约束优化（组合求解）+ 时序预测（社区需求）。详见 `spec/产品规格文档.md` 第 7 章。
